06 — App icon spec
PropertyValueNotes
Concept Weapon card, domain colors Warzone loadout card shape. 4 domain slots in 2x2 grid, each with its own accent color.
Background #0d0d0d Near-black, not pure black. Gives depth without crushing contrast.
Card border #252525, 1px Outer weapon card frame. Subtle, gives the card shape its structure.
Header stripe #161616 Top 18px of the card. Small white label dot at left edge.
Slot: Training bg #080c14, stroke #4ea8de Top-left. Dumbbell silhouette. Blue accent on fill and stroke.
Slot: Nutrition bg #081410, stroke #3cc8a0 Top-right. Plate and fork silhouette. Teal accent.
Slot: Budget bg #140e04, stroke #e8a93a Bottom-left. Wallet silhouette. Amber accent.
Slot: Schedule bg #111118, stroke #9898a8 Bottom-right. Calendar silhouette. Slate accent.
Slot border radius 5px Inside slots only. Outer card uses 8px rx.
Slot stroke width 0.9px at 130px viewBox Increase proportionally for larger exports (1.5px at 1024px).
Export sizes 1024px, 180px, 120px, 87px, 60px, 40px, 29px, 20px iOS and Android full set. At sizes below 40px, inner silhouettes are invisible — only slot color blocks remain, which is intentional.
Format SVG source + PNG exports Keep SVG as source of truth. Export PNG at 3x for each required size.
Status Working direction, not final Icon to be refined iteratively. Slot silhouettes may evolve.
Note for agent: The icon SVG source is embedded in this document. The viewBox is 0 0 130 130. Do not change this value. When implementing in React Native, use the SVG directly via react-native-svg. For the Expo app icon, export a 1024x1024 PNG from the SVG at the correct dimensions. The four domain colors in the icon are the same tokens used throughout the app UI, so a change to any domain color should propagate to both the icon slots and all in-app usage automatically via the token system above.

08 — Implementation notes
AreaDecision
FrameworkExpo (React Native) for mobile. FastAPI for backend.
NavigationBottom tab bar, 4 tabs matching the 4 domains. Each tab uses its domain color as active indicator.
ThemeDark mode only in v1. System tokens defined as constants in a single theme.ts file. No light mode until v2.
Domain colorsDefined once in theme.ts. Never hardcoded inline. Imported everywhere via the token name, e.g. colors.training.
Border styleAll borders are 0.5px or 1px, #202020. No shadows. Elevation conveyed by background lightness only.
ChartsVictory Native or Recharts (web). Colored by domain. No legends on simple charts.
Icons (UI)Lucide React Native. Consistent 20px size, strokeWidth=1.5.
FontsSystem font stack only in v1. No custom font loading. Avoids FOUT and reduces bundle.
AI assistantRAG pipeline on FastAPI. ChromaDB or pgvector. Context window includes user's last 7 days of data across all domains.
State managementZustand for client state. React Query for server state and caching.
For the VSCode agent: Start with the theme file and navigation shell before any feature screens. Get the bottom tab bar working with the 4 domain colors first — that establishes the entire visual language for every screen that follows. The design system above is the source of truth. Any color, spacing, or radius value not in this document should be derived from the nearest token above, never invented.
